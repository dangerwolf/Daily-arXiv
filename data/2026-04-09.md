*Susie Xi Rao,Johannes Rausch,Peter Egger,Ce Zhang*

Main category: cs.CV

TL;DR: TableParser能够高精度地解析原生PDF和扫描图像中的表格结构和内容，并通过领域自适应技术进行优化。同时，研究者还开发了TableAnnotator和ExcelAnnotator，构建了基于电子表格的弱监督机制和表格解析流水线，并开源了相关资源。


<details>
  <summary>Details</summary>
Motivation: 表格作为一种普遍的数据存储结构，对解析其结构和提取内容的需求在许多应用中都非常重要。然而，目前处理PDF、图片、表格、CSV等不同格式的表格解析方法在精度和适应性上仍有提升空间。

Method: 本文提出了一种名为TableParser的系统，能够解析原生PDF和扫描图像中的表格。该系统利用了领域自适应技术来提高解析精度。此外，研究者还开发了TableAnnotator和ExcelAnnotator，建立了一个基于电子表格的弱监督机制和一个用于表格解析的流水线。

Result: 通过大量的实验证明，TableParser在解析表格结构和内容方面表现出高精度，并且领域自适应技术能够有效提升其性能。TableAnnotator和ExcelAnnotator的结合为表格解析提供了一种新的弱监督方法。

Being able to parse table structures and extract content bounded by these structures is of high importance in many applications. In this paper, we devise TableParser, a system capable of parsing tables in both native PDFs and scanned images with high precision. We have conducted extensive experiments to show the efficacy of domain adaptation in developing such a tool. Moreover, we create TableAnnotator and ExcelAnnotator, which constitute a spreadsheet-based weak supervision mechanism and a pipeline to enable table parsing. ### [2] [The Tool Decathlon: Benchmarking Language Agents for Diverse, Realistic, and Long-Horizon Task Execution](https://arxiv.org/abs/2510.25726)
*Junlong Li,Wenshuo Zhao,Jian Zhao,Weihao Zeng,Haoze Wu,Xiaochen Wang,Rui Ge,Yuxuan Cao,Yuzhen Huang,Wei Liu,Junteng Liu,Zhaochen Su,Yiyang Guo,Fan Zhou,Lueyang Zhang,Juan Michelini,Xingyao Wang,Xiang Yue,Shuyan Zhou,Graham Neubig,Junxian He*

Main category: cs.CL

TL;DR: Tool Decathlon (Toolathlon) is a new benchmark for evaluating language agents on complex, multi-step workflows across diverse real-world applications, featuring 32 apps and 604 tools. It offers realistic environment states and execution-based evaluation for 108 tasks, with average lengths of 20 turns. Current state-of-the-art models show significant limitations, with the best achieving only a 38.6% success rate.


<details>
  <summary>Details</summary>
Motivation: Existing language agent benchmarks are insufficient for evaluating real-world performance due to narrow domains, simplified tasks, and lack of diversity and long-horizon complexity. A new benchmark is needed to address these limitations.

Method: Introduced Tool Decathlon (Toolathlon), a benchmark comprising 32 software applications and 604 tools, including everyday and professional platforms. It features realistic initial environment states derived from real software and includes 108 manually sourced or crafted tasks requiring multi-app interaction over an average of 20 turns. Evaluation is execution-based using dedicated scripts.

Result: Comprehensive evaluation of state-of-the-art models revealed significant shortcomings. The best model, Claude-4.5-Sonnet, achieved a 38.6% success rate with an average of 20.2 tool-calling turns, while the top open-weights model, DeepSeek-V3.2-Exp, reached 20.1% success.

Conclusion: Toolathlon serves as a comprehensive benchmark to drive the development of more capable language agents for real-world, long-horizon task execution, multi-app task execution, highlighting the current limitations of existing models.

Abstract: Real-world language agents must handle complex, multi-step workflows across diverse Apps. For instance, an agent may manage emails by coordinating with calendars and file systems, or monitor a production database to detect anomalies and generate reports following an operating manual. However, existing language agent benchmarks often focus on narrow domains or simplified tasks that lack the diversity, realism, and long-horizon complexity required to evaluate agents' real-world performance. To address this gap, we introduce the Tool Decathlon (dubbed as Toolathlon), a benchmark for language agents offering diverse Apps and tools, realistic environment setup, and reliable execution-based evaluation. Toolathlon spans 32 software applications and 604 tools, ranging from everyday platforms such as Google Calendar and Notion to professional ones like WooCommerce, Kubernetes, and BigQuery. Most of the tools are based on a high-quality set of Model Context Protocol (MCP) servers that we may have revised or implemented ourselves. Unlike prior works, which primarily ensure functional realism but offer limited environment state diversity, we provide realistic initial environment states from real software, such as Canvas courses with dozens of students or real financial spreadsheets. This benchmark includes 108 manually sourced or crafted tasks in total, requiring interacting with multiple Apps over around 20 turns on average to complete. Each task is strictly verifiable through dedicated evaluation scripts. Comprehensive evaluation of SOTA models highlights their significant shortcomings: the best-performing model, Claude-4.5-Sonnet, achieves only a 38.6% success rate with 20.2 tool calling turns on average, while the top open-weights model DeepSeek-V3.2-Exp reaches 20.1%. We expect Toolathlon to drive the development of more capable language agents for real-world, long-horizon task execution.

### [3] [TableQuery: Querying tabular data with natural language](https://arxiv.org/abs/2202.00454)
*Abhijith Neil Abraham,Fariz Rahman,Damanpreet Kaur*

Main category: cs.CL

TL;DR: TableQuery是一个使用预训练的深度学习模型来查询表格数据的工具，解决了现有方法需要将整个表输入模型的内存限制和实时更新问题。


<details>
  <summary>Details</summary>
Motivation: 现有表格数据问答方法存在局限性，需要将整个表输入模型，不适用于包含大量数据或实时更新的真实场景。

Method: 使用预训练的、针对自由文本问答的深度学习模型，将自然语言查询转换为可直接在数据库或电子表格上运行的结构化查询。

Result: 避免了将整个数据加载到内存的需要，也无需在数据更新时序列化数据库。模型易于更新，可替换为性能更优的新模型。

Conclusion: TableQuery提供了一种高效、灵活的表格数据查询解决方案，无需重新训练，并且能够适应实时更新的数据。

Abstract: This paper presents TableQuery, a novel tool for querying tabular data using deep learning models pre-trained to answer questions on free text. Existing deep learning methods for question answering on tabular data have various limitations, such as having to feed the entire table as input into a neural network model, making them unsuitable for most real-world applications. Since real-world data might contain millions of rows, it may not entirely fit into the memory. Moreover, data could be stored in live databases, which are updated in real-time, and it is impractical to serialize an entire database to a neural network-friendly format each time it is updated. In TableQuery, we use deep learning models pre-trained for question answering on free text to convert natural language queries to structured queries, which can be run against a database or a spreadsheet. This method eliminates the need for fitting the entire data into memory as well as serializing databases. Furthermore, deep learning models pre-trained for question answering on free text are readily available on platforms such as HuggingFace Model Hub (7). TableQuery does not require re-training; when a newly trained model for question answering with better performance is available, it can replace the existing model in TableQuery.

### [7] [Spreadsheet computing with Finite Domain Constraint Enhancements](https://arxiv.org/abs/2203.10944)
*Ezana N. Beyenne*

Main category: cs.AI

TL;DR: 本文提出了一种将有限约束求解器与电子表格计算范式相结合的框架，以解决约束满足问题。


<details>
  <summary>Details</summary>
Motivation: 电子表格应用易于使用且有用，但仅限于类似簿记的任务，因为它们是单向数据流。本文旨在通过扩展电子表格计算范式来克服这一限制，以解决约束满足问题。

Method: 该框架允许将有限域或指定单元格之间关系的约束附加到电子表格中的单元格。该框架提供了一个约束求解接口，并通过提供一组特定于电子表格的约束来进一步增强电子表格计算范式，以帮助控制大型电子表格应用程序实现的伸缩性。

Result: 通过示例演示了扩展的电子表格范式的可用性和实用性。

Conclusion: 所提出的框架有效地将有限约束求解器集成到电子表格计算范式中，从而扩展了电子表格的功能，使其能够解决约束满足问题。

Abstract: Spreadsheet computing is one of the more popular computing methodologies in today's modern society. The spreadsheet application's ease of use and usefulness has enabled non-programmers to perform programming-like tasks in a familiar setting modeled after the tabular "pen and paper" approach. However, spreadsheet applications are limited to bookkeeping-like tasks due to their single-direction data flow. This thesis demonstrates an extension of the spreadsheet computing paradigm in overcoming this limitation to solve constraint satisfaction problems. We present a framework seamlessly incorporating a finite constraint solver with the spreadsheet computing paradigm. This framework allows the individual cells in the spreadsheet to be attached to either a finite domain or a constraint specifying the relationship among the cells. The framework provides an interface for constraint solving and further enhances the spreadsheet computing paradigm by providing a set of spreadsheet-specific constraints that will aid in controlling the scalability of large spreadsheet applications implementations. Finally, we provide examples to demonstrate the usability and usefulness of the extended spreadsheet paradigm.
  Keywords: Spreadsheet computing, Constraint Logic Programming, Constraint satisfaction, Domain-Specific language, Excel, SWI Prolog, C#

</details>
